# v0.1.0 发布：zhenyi 解决方案层首次开源

> 从基础库到完整解决方案，一个命令跑起分布式 IM。

## 一句话定位

**zhenyi** 是面向实时业务（游戏、IM、IoT 等）的高性能 Go 解决方案，提供 **Actor 模型 + 统一网关（`zgate`）+ 分布式协作能力**，帮助你更快搭建可观测、可扩展的实时后端。

## 这次发布有什么？

### ✅ Actor 运行时（`zactor`）
- 单 Actor 单 mailbox（面向并发的消息驱动处理）
- 消息处理、Tick、RPC、Dispatcher 扩展
- 为高吞吐场景提供稳定的消息发送/回执路径

### ✅ 统一接入层（`zgate`，可选 `zhttp`）
- 支持 TCP/WS/KCP 长连接
- 可挂载到 Gate 的处理链路，业务代码复用
- 可选 HTTP 服务能力（对应 `zhttp`）

### ✅ 服务发现与路由（`zdiscovery` + `zroute`）
- Etcd / Noop 等发现实现
- 远程路由策略包含一致性哈希（HRW / RendezvousHash）等选项

### ✅ 可观测性（`zmetrics` / `zmonitor` / `ztrace`）
- 指标导出与健康探针
- 结构化监控快照、traceparent 解析与链路透传

### ✅ 外部总线集成（`znats` / `zbus`）
- NATS / 总线抽象，用于跨进程消息协作

## 稳定性验证（已观测阶段，压测仍在进行中）

> 注：以下数据来自一次本地压测的观测窗口（示例数据，用于说明指标含义与量级；不同环境会有差异）。

- 场景：500 连接 / 10K QPS（单机）
- 延迟（RTT）：P50 约 `5.5ms`，P99 约 `34ms`
- 内存：约 `28MB`
- GC：暂停占比 `<0.1%`

完整长时段统计、以及更细的 P99/P999 分位与波动情况，建议在压测结束后补充最终报告。

## 快速体验（几分钟）

### 1) 启动服务端（Gate + IM Actor）
```bash
go run ./examples/im_single_demo
```

### 2) 启动客户端（交互式聊天）
```bash
go run ./examples/im_single_client \
  -addr 127.0.0.1:8001 \
  -user 10001 \
  -nick alice \
  -room lobby \
  -msgLogin 1 \
  -msgJoin 2 \
  -msgLeave 3 \
  -msgSend 4
```

启动后在客户端控制台输入文本回车即可发送消息；支持 `/join`、`/leave`、`/quit` 等交互命令。

## 适用场景

- 游戏服务器：房间、匹配、战斗等实时交互
- 物联网平台：设备连接、指令下发与回执
- 实时 IM / 社交应用：会话、房间、广播与在线状态
- 工程化/合规项目：AGPL + 商业双授权下的可控交付

## 已知限制

- 当前文档与示例仍在持续完善；部分高级场景以模块 README 与源码行为为准
- 不同传输协议（TCP/WS/KCP）在极端网络条件下的性能表现需按业务场景压测验证
- 分布式部署依赖外部组件（如发现服务、消息总线）时，需自行规划高可用拓扑

## 升级 / 迁移建议

- 从基础层能力迁移时，优先保持 `ziface` 接口契约不变，按模块逐步替换
- 先完成消息模型与路由规则对齐，再引入跨进程发现与总线能力
- 发布前执行最小回归：连接建立、消息收发、路由回退、监控暴露
