### 2.3 场景 A 小结

> **2000 连接 × Echo 回环 × 100ms 间隔 × 7 天连续运行**
> - 测试链路：Client → Gate → IM → Gate → Client，每条消息完整往返
> - 累计处理 **120.96 亿**次 echo 回环（服务端实际消息吞吐 **241.92 亿次**，去+回各算一次）
> - QPS 稳定 **20,000 msg/s**（客户端视角）/ **40,000 msg/s**（服务端视角）
> - P50 延迟 **0.11ms**，P99 延迟 **1.3ms**，7 天无退化
> - 每秒约 0.25 次 GC，但每次暂停仅 **0.3μs**，7 天总暂停 **46 秒**（占总时间 0.0076%）
> - 内存无泄漏，goroutine 无泄漏
> - 客户端未回包数（sent-recv）**23 条 / 120.96 亿** = **1.9×10⁻⁸**（该差值可能包含退出时未统计完的尾部回包）

## 四、两场压测对比分析

| 维度 | 场景 A（7 天长稳） | 场景 B（1 小时高压） |
|------|-------------------|---------------------|
| 测试链路 | Echo 回环（Client↔Gate↔IM） | Echo 回环（Client↔Gate↔IM） |
| 连接数 | 2000 | 500 |
| 发送间隔 | 100ms | 1ms |
| 客户端 Echo QPS | 20,000 回环/s | 83,500 回环/s |
| 服务端实际消息吞吐 | 40,000 msg/s（去+回） | 148,000 ~ 171,000 msg/s（去+回） |
| Actor 邮箱吞吐 | 80,000 次/s（4次/回环） | ~334,000 次/s（4次/回环） |
| RTT P50 | **0.11 ~ 0.19 ms** | 8.7 ~ 13 ms |
| RTT P99 | **1.27 ~ 1.52 ms**（99.9% < 1.5ms） | 47 ~ 88 ms |
| RTT Max | **142ms**（全局峰值，04-09 04:00 cron 触发），72% 采样窗口 2-10ms | ~80ms |
| GC 频率 | ~0.25 次/秒（每 5s 间隔均值 1.28 次） | 每 5s 间隔 1-6 次 |
| GC 总暂停 | 46 秒（占总时间 0.0076%） | 每次 0.3μs |
| 内存分配 | 69 ~ 125 MB | 未记录（初始为 0） |
| Goroutines | 25（7 天无泄漏） | 25 |
| 消息丢失率 | 1.9×10⁻⁸ | 1.95×10⁻⁵ |
| 总消息量 | **120.96 亿** | **3.01 亿** |

### 关键洞察

1. **Echo 回环含金量高于纯 pubsub**：每条消息经历 Client→Gate→IM→Gate→Client 四跳 Actor 邮箱投递。客户端 20K QPS 意味着 Actor 邮箱实际吞吐 80K 次/s（场景 A），83.5K echo/s 意味着 ~334K 次/s（场景 B）。

2. **延迟与负载强相关**：场景 B 发送频率是场景 A 的 100 倍，P50 延迟从 0.15ms 上升到 10ms，P99 从 1.3ms 上升到 60ms。符合 Actor 邮箱排队的预期行为。场景 A 的 RTT Max 尾部延迟（每 5s 窗口内最大值）分布为 2-10ms 占 72%、10-50ms 占 21%，是 10 万次请求中的统计尾部，不反映典型延迟。

6. **凌晨 cron 任务产生系统性延迟毛刺**：每天 04:00 的 `systemctl restart update-motd` 在 7 天中 **7/7 天无一例外** 触发了延迟异常——Samples 从 ~80K 暴跌至 3K-35K，P99 飙升至 12-88ms，Max 最高 142ms。影响持续 20-40 秒后恢复。全局 4 次 100ms+ 峰值中 3 次为 cron 导致，1 次为 SSH 登录操作导致。生产环境应避免在业务进程所在机器运行非必要的定时任务。

3. **GC 表现优异**：7 天 120 亿消息，每秒 GC 约 0.25 次但每次仅 0.3μs，总暂停 46 秒（占总时间 0.0076%）。说明 zhenyi 运行时的堆极小、分配策略高度优化。

4. **零泄漏**：25 个 goroutine 7 天无增长，内存无膨胀，这是生产级运行时的基本要求。

5. **消息几乎不丢**：场景 A 丢失率 1.9×10⁻⁸（百亿分之二），场景 B 稍高为 1.95×10⁻⁵（万分之二），后者是高压下少量尾部消息在断开时未完全确认。
